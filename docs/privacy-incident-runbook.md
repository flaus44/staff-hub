# Privacy incident runbook (staff & co-design data)

**Privacy contact:** mentors@flaus.com.au  
**Section 8 data:** Daniel Ross (`contactDataViewer`)

## When to report (within 24 hours)

- Lost or stolen device with an open Session Capture draft
- Session Capture or contact export sent to the wrong person
- Unauthorised access to participant data
- Suspected ransomware or database compromise
- Facilitator accidentally recorded excessive PII in free text

## Steps

1. **Contain** — revoke access, delete erroneous export, lock affected accounts.
2. **Report** — email mentors@flaus.com.au with: what happened, when, data classes affected (A/B/C/D), approximate number of people.
3. **Assess** — privacy lead decides if OAIC NDB notification is required (30-day window).
4. **Notify** — affected individuals if required; document all communications.
5. **Record** — create `/incidents` entry with tag `privacy_breach` (no participant names in description unless necessary).

## Do not

- Investigate abuse disclosures in the form — use distress protocol and escalate to FLAUS.
- Email participant data to personal accounts.
- Screenshot Section 8 contact details.
